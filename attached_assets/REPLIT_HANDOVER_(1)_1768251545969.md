# Betrayal Game - Replit Handover Document

## 🎮 Project Overview

**Betrayal Game** is a multiplayer social deduction game inspired by "The Traitors" TV show. Players are secretly assigned as either Traitors or Faithful, and must vote to eliminate players while Traitors secretly murder at night.

**Current Status:** Week 4 Complete - Multiple Traitors Feature Implemented ✅

## 🔧 Development Guide

### Running Locally
```bash
npm run dev          # Start development server
npx tsc --noEmit     # Check TypeScript errors
```

### Key Files to Understand

#### `src/game/types.ts`
Defines all TypeScript interfaces and events:
- `GameState`: Main game state object
- `Player`: Player information
- `C2SEvent`: Client-to-server events
- `S2CEvent`: Server-to-client events

#### `src/game/manager.ts`
Core game logic functions:
- `createGame()` / `addPlayer()`: Game setup
- `assignRoles()`: Role distribution (Week 4 scaling)
- `submitVote()` / `banishPlayer()`: Voting mechanics
- `submitMurder()` / `resolveMurder()`: Night phase (Week 4 coordination)
- `checkWinCondition()`: Game end detection
- **Week 4 helpers:**
  - `getTraitorIds()`: Returns all traitor IDs
  - `getAliveTraitorCount()`: Count alive traitors
  - `getMurderVoteProgress()`: Track murder votes

#### `src/index.ts`
WebSocket server and event routing:
- Connection handling
- Event parsing and validation
- Broadcasting to players
- Per-player state management

### Adding New Events

**1. Define event types in `types.ts`:**
```typescript
// Client-to-Server
export type C2SEvent = 
  | { type: 'C2S_NEW_ACTION'; payload: { data: string } }
  | ... existing events

// Server-to-Client  
export type S2CEvent =
  | { type: 'S2C_NEW_RESPONSE'; payload: { result: string } }
  | ... existing events
```

**2. Add game logic in `manager.ts`:**
```typescript
export function newGameFunction(game: GameState, data: string): GameState {
  // Your logic here
  return {
    ...game,
    // Updated state
  };
}
```

**3. Add handler in `index.ts`:**
```typescript
if (event.type === 'C2S_NEW_ACTION') {
  const updatedGame = game.newGameFunction(gameState, event.payload.data);
  games.set(currentSessionId, updatedGame);
  
  broadcastToSession(currentSessionId, {
    type: 'S2C_NEW_RESPONSE',
    payload: { result: 'success' }
  });
  return;
}
```

## 🐛 Known Issues & Limitations

### Current Limitations
1. **No persistence:** Games lost on server restart
2. **No reconnection:** Disconnected players lose session
3. **No UI:** Command-line/WebSocket testing only
4. **No authentication:** Anyone can send any event
5. **No rate limiting:** Potential for spam

### TypeScript Configuration
- Using strict mode with `noUncheckedIndexedAccess`
- Requires explicit `.js` extensions in imports (ESM)
- All functions are strongly typed

## 📚 API Reference

### Client-to-Server Events

| Event | Payload | Description | Phase Required |
|-------|---------|-------------|----------------|
| `C2S_CREATE_GAME` | `{ playerName: string }` | Create new game | - |
| `C2S_JOIN_GAME` | `{ sessionId: string, playerName: string }` | Join existing game | LOBBY |
| `C2S_START_GAME` | `{}` | Start game (host only) | LOBBY |
| `C2S_ASSIGN_ROLES` | `{}` | Assign roles to players | ROLE_ASSIGN |
| `C2S_START_VOTING` | `{}` | Begin voting phase | ROUNDTABLE |
| `C2S_SUBMIT_VOTE` | `{ targetId: string }` | Vote to banish player | VOTING |
| `C2S_REVEAL_VOTES` | `{}` | Show all votes | VOTING |
| `C2S_BANISH_PLAYER` | `{}` | Execute banishment | VOTE_REVEAL |
| `C2S_CHECK_WIN` | `{}` | Check win conditions | BANISH_REVEAL |
| `C2S_START_NIGHT` | `{}` | Begin night phase | CHECK_WIN |
| `C2S_SUBMIT_MURDER` | `{ targetId: string }` | Traitor votes to murder | NIGHT |
| `C2S_RESOLVE_MURDER` | `{}` | Execute murder | NIGHT |
| `C2S_START_MORNING` | `{}` | Begin morning reveal | MORNING |
| `C2S_CONTINUE_TO_DAY` | `{}` | Return to roundtable | MORNING |

### Server-to-Client Events

| Event | Payload | Description |
|-------|---------|-------------|
| `S2C_GAME_CREATED` | `{ sessionId, playerId, playerName }` | Game created successfully |
| `S2C_PLAYER_JOINED` | `{ players: Player[] }` | Player joined game |
| `S2C_ROLES_ASSIGNED` | `{ phase }` | Roles assigned to all |
| `S2C_ROLE_REVEAL` | `{ role, phase, traitorIds? }` | Your role (traitors get IDs) |
| `S2C_VOTING_STARTED` | `{ phase }` | Voting has begun |
| `S2C_VOTE_SUBMITTED` | `{ voterId }` | A vote was submitted |
| `S2C_VOTES_REVEALED` | `{ votes, phase }` | All votes shown |
| `S2C_PLAYER_BANISHED` | `{ banishedPlayerId, name, role, phase }` | Player eliminated |
| `S2C_GAME_END` | `{ winner, phase, remainingTraitors, remainingFaithful }` | Game over |
| `S2C_CONTINUE_GAME` | `{ phase, currentRound }` | Game continues |
| `S2C_NIGHT_STARTED` | `{ phase, currentRound, aliveTraitorCount }` | Night phase begun |
| `S2C_MURDER_SUBMITTED` | `{ voterId, votesReceived, votesNeeded }` | Murder vote submitted |
| `S2C_MURDER_RESOLVED` | `{ murderedPlayerId, name, phase }` | Player murdered |
| `S2C_MORNING_STARTED` | `{ phase, lastMurderedPlayerId?, name? }` | Morning reveal |
| `S2C_ERROR` | `{ message }` | Error occurred |

## 🎨 Design Considerations

### Game Balance (Week 4)
The multiple traitor system maintains balance:
- **Small games (5-9):** 1 traitor vs 4-8 faithful = 11-20% traitors
- **Medium games (10-14):** 2 traitors vs 8-12 faithful = 14-20% traitors
- **Large games (15-19):** 3 traitors vs 12-16 faithful = 16-19% traitors
- **Max games (20-22):** 4 traitors vs 16-18 faithful = 18-20% traitors

Murder voting strategy:
- **Unanimous:** Traitors coordinate = guaranteed elimination
- **Split votes:** Traitors disagree = random target (risk for traitors)

### UX Improvements Needed
1. **Clear phase indicators:** Players should know what to do
2. **Player cards:** Visual representation of alive/dead/role
3. **Vote history:** Track who voted for whom
4. **Murder reveal:** Dramatic reveal of victim in morning
5. **Win screen:** Show all roles, who won, statistics
